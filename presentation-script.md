## Section 01: 环境配置

- 首先，我们从最基础的开始 - 怎么配置环境，开始使用 Claude

### 第一步：访问 Claude

- Claude 可以通过网页端 claude.ai 直接访问，也可以下载桌面端和手机端
- **桌面端**支持 macOS 和 Windows
- **手机端**支持 iOS 和 Android
- 需要注意的是，国内访问需要 VPN
- 浏览器推荐用 Chrome，体验最好

### 第二步：选择订阅方案

- Claude 有四种订阅方案，我简单讲一下区别：
  - **Free（免费版）**：基础功能，每天 30-100 条消息，适合体验
  - **Pro（专业版）**：每月 20 美金，使用量是免费版的 5 倍，解锁所有模型、Claude Code、Projects 等核心功能
  - **Max（旗舰版）**：每月 100-200 美金，有持久记忆、优先体验新功能
  - **Team（团队版）**：每人每月 25-125 美金，适合团队协作
- **建议**：如果是个人使用，先从 Pro 版开始，功能完整，性价比最高

### 第三步：了解模型选择

- Claude 目前有三个主力模型：
  - **Haiku 4.5** - 极速模型，响应最快、成本最低，适合简单的高频任务
  - **Sonnet 4.6** - 均衡模型，智能和速度的最佳平衡，**日常 90% 的任务用它就够了**
  - **Opus 4.6** - 深度推理模型，推理能力最强，适合复杂分析和 Agent 任务
- **记住一个简单原则**：日常用 Sonnet，复杂推理用 Opus

### 第四步：开始第一次对话

- 配置好之后，就可以开始对话了
- 核心原则：**输出质量取决于输入质量**
- 一个好的提示词应该包含四个要素：
  - **角色设定** - 告诉 AI 它应该扮演什么角色
  - **背景信息** - 提供必要的上下文
  - **具体任务** - 明确你要它做什么
  - **输出格式** - 说清楚你要什么格式的结果

## Section 02: Claude 简介

- 接下来，我来解释一下为什么我们选择 Claude 作为主力 AI 工具

### 为什么选 Claude？

- 首先说一个客观数据：在 **Chatbot Arena** 这个全球最权威的 AI 模型盲测排行榜上，Claude 长期处于第一梯队
- Chatbot Arena 的评测方式是让真实用户进行 **双盲投票** - 用户不知道自己在跟哪个模型对话，纯粹根据回答质量来打分
- 这意味着排名是完全客观的，不是哪家公司自己说自己好
- Claude 在 **编程、推理、写作** 这三个核心能力上表现尤为突出
- 这也是 Mooov 团队推荐 Claude 作为首选 AI 工具的原因

### 可用模型概览

- 来详细看一下三个模型的定位和适用场景：

#### Haiku 4.5 - 极速

- **适用场景**：高频简单任务、客服对话、数据预处理
- 响应速度最快、延迟最低、成本最低
- API 价格：输入 $1 / 输出 $5（每百万 Token）
- 把它理解为一个 **"快速执行者"**

#### Sonnet 4.6 - 均衡 (推荐)

- **适用场景**：日常工作、写作、分析、编程、翻译
- 智能和速度的最优平衡点
- 支持 Extended Thinking 和 Adaptive Thinking - 能根据任务复杂度自动调整思考深度
- API 价格：输入 $3 / 输出 $15（每百万 Token）
- **这是 90% 场景的首选模型**

#### Opus 4.6 - 深度推理

- **适用场景**：复杂推理、Agent 任务、架构设计、深度研究
- 最强的推理能力
- 支持最高 128K tokens 输出
- 专为 Agent 工作和高难度编程设计
- API 价格：输入 $5 / 输出 $25（每百万 Token）

### Token 与成本

- 说到成本，有一个概念需要理解 - **Token**
- Token 是 AI 模型处理文本的基本单位
- **最大上下文窗口**：200K+ tokens，相当于一本 15 万字的中文小说
- 目前 1M beta 版本可以处理约 340 万 Unicode 字符
- **换算规则**：大约 1000 个中文字 = 1500 个 tokens
- 理解 Token 有助于你控制成本、优化对话效率

## Section 03: AI 模型对比

- 接下来是一个很实用的板块 - 不同场景下，到底该用哪个 AI 工具
- 我们对比了六个核心场景

### 1. 文档/内容创作 (Content Creation)

- **推荐工具：Claude**
- 适用场景：营销文案、报告、邮件、技术文档、社交媒体内容、文章
- **关键技巧**：用中文写提示词就出中文内容，用英文写就出英文内容。Claude 完全理解中文社交媒体的语气和风格
- 举个例子，你可以这样写提示词：

> 你是一位资深内容策略师。请为我们公司的新产品发布撰写一篇小红书种草笔记。要求：标题20字以内使用二极管标题法，正文700字口语化表达，每段使用emoji，融入"宝藏""绝绝子"等热门词汇，文末带5-10个话题标签。

- 这种程度的细节指导，会让输出质量大幅提升

### 2. 内容搜索 / Research

- **推荐工具：Claude**
- 适用场景：市场调研、竞品分析、学术文献综述、行业趋势、政策解读
- **核心优势**：200K+ token 的上下文窗口（1M beta），可以在一次会话中处理相当于一本 15 万字中文小说的信息量
- 最擅长的是：**跨文档分析、信息综合、深度研究**
- 比如你可以说：

> 分析企业级AI生产力工具的竞争格局。从市场份额、核心功能、定价模式、目标客户四个维度对比前5家竞品。用表格呈现对比结果，然后为新进入者提供SWOT分析。

### 3. 图片生成 (Image Generation)

- **推荐工具：Google Imagen（不是 Claude）**
- Claude 本身不直接生成图片，但它非常擅长 **写精确的图片生成提示词**
- 适用场景：产品图、社交媒体配图、品牌视觉、PPT 插图、营销海报
- Google Imagen 系列和国内的 Seedance 2.0 在图片生成上都是世界级水准
- 最佳工作流：用 Claude 写 prompt，然后在图片生成工具中使用

### 4. 编程 (Programming)

- **推荐工具：Claude / ChatGPT Codex**
- 适用场景：代码生成、Bug 修复、代码审查、架构设计、测试编写、数据库设计、API 开发
- **Claude Code 的核心优势**：它能独立理解整个代码库、编辑多个文件、运行测试、执行命令
- Multi-Agent 模式可以实现 **5-10 倍的效率提升**
- 这不是代码补全工具 - 这是一个能独立思考、规划、执行的 AI 软件工程师

### 5. 数据处理 (Data Processing)

- **推荐工具：Claude**
- 适用场景：Excel 数据清洗、CSV 分析、可视化建议、报告生成、数据转换、统计分析
- **核心优势**：Claude 可以直接处理上传的 Excel/CSV 文件
- Claude Cowork（桌面端 Agent）可以直接生成带有效公式的 Excel 文件 - VLOOKUP、条件格式、数据分析图表，都能自动生成

### 6. 学习与教育辅助

- **推荐工具：Claude / Gemini**
- 适用场景：概念讲解、知识测试、学习路径规划、论文辅导、快速进入新领域
- Claude 擅长 **分层讲解复杂概念**，支持苏格拉底式教学法，能根据学习者水平调整解释深度
- Gemini 默认支持 1M 上下文窗口，适合处理大量学习材料

## Section 04: 提示词技巧

- 这是今天非常核心的一个板块 - **Prompt Engineering**
- 有研究表明，**结构化中文提示词的有效率达到 92.3%**
- 好的提示词是 AI 协作的基础

### 实际演示

- 给大家看一个实际的例子。我给 Claude 这样一个提示词：

> 你是一位拥有15年经验的高级产品经理。我们公司准备发布一款面向中小企业的SaaS协作工具。请分析我们的产品上市策略，并给出3条关键建议。要求：每条建议包含具体行动项和预期效果。

- Claude 的回答是结构化的，从三个维度分析：
  - **精准定位差异化** - 聚焦"中小企业专属"标签，避免与大厂正面竞争
  - **种子用户冷启动** - 招募 50-100 家 Beta 测试企业
  - **建立反馈闭环** - 形成"收集-分析-迭代-验证"的持续优化循环
- 每条建议都有具体行动项和预期效果，这就是好提示词带来的好输出

### 四大核心提示词技巧

#### 1. 角色扮演 (Role Playing)

- 给 AI 分配一个专业角色，让它以领域专家的身份回答问题
- **角色定义越具体，输出质量越高** - 包含经验年限、专长领域、工作风格
- 研究表明：基于角色的提示词能平均 **提升 40% 的输出质量**
- 示例：

> 你是一位拥有20年经验的品牌策略顾问，曾服务过多家金融科技公司。你的风格务实直接，擅长用数据说话。请分析我们公司的品牌定位，并给出优化建议。

#### 2. 结构化提示 (Structured Prompts)

- 用清晰的格式和标签来组织提示词
- 好的结构化提示包含：**背景 (Context)、任务 (Task)、约束条件 (Constraints)、输出格式 (Format)**
- 用分隔符（---、```、""）来区分不同部分
- 示例：

```
# 背景
我们是一家B2B金融科技公司

# 任务
撰写一份季度市场分析报告

# 要求
- 3000字以内
- 包含数据图表描述
- 使用正式商务语气

# 输出格式
Markdown格式，含标题分级
```

#### 3. 思维链 (Chain of Thought)

- 引导 AI 一步一步思考，展示推理过程
- 加入"让我们一步步思考"这样的引导语，或者把思考拆成阶段
- **最适合**：复杂决策、数据分析、需要推理链条的问题诊断
- 示例：

> 请一步步分析这个商业决策：
> 第一步：列出当前市场状况和关键数据
> 第二步：分析3种可选方案的优劣势
> 第三步：评估每种方案的风险和收益
> 第四步：给出你的推荐方案和实施路线图

#### 4. 示例引导 (Few-shot Learning)

- 提供 input-output 示例，展示你期望的格式和风格
- 这是让 AI 精准匹配输出预期的 **最有效方式**
- 通常 2-3 个覆盖不同场景的示例就够了
- 示例：

> 请按以下格式分析公司：
>
> 示例输入：苹果公司
> 示例输出：行业：消费电子+软件生态 / 核心优势：品牌溢价+闭环生态 / 风险：创新放缓、中国市场份额下降
>
> 示例输入：特斯拉
> 示例输出：行业：新能源汽车+能源 / 核心优势：技术领先+品牌效应 / 风险：竞争加剧、产能扩张压力
>
> 请分析：[你的目标公司]

## Section 05: 高阶功能

- 接下来是今天的重头戏 - Claude 的高阶功能
- 从基础聊天到工程级应用，Claude 不只是一个对话工具，它是一个 **完整的 AI 协作平台**

### Feature 1: Claude Code - 工程化 AI 的终极形态

- **Claude Code** 是 Anthropic 的 Agent 级编程工具，在终端中运行
- 它能独立理解整个代码库、编辑文件、运行命令
- **这不是代码补全 - 这是一个能独立思考、规划、执行的 AI 软件工程师**
- 支持 VS Code、JetBrains、桌面端、终端

#### 核心能力一：Claude Code + Figma - 设计到代码

- Figma MCP Server 让 Claude Code 可以直接读取设计稿，理解设计意图、组件结构、样式规范
- 2026年2月17日 Figma 发布了 "Code to Canvas" 功能 - Claude Code 生成的 UI 可以反向转化为可编辑的 Figma 设计
- 这实现了真正的 **双向工作流**：Figma 设计 -> Claude Code 生成代码 -> 运行中的 UI -> Figma 反馈 -> 设计师调整 -> 代码更新

#### 核心能力二：Sub-Agents / Agent Teams - 多 Agent 协作

- Claude Code 可以生成多个 Sub-Agent，每个专注特定任务，**并行工作**
- Team Lead Agent 负责任务分解和协调，Worker Agent 执行具体任务
- 每个 Agent 可以独立配置：定制提示词、工具权限、模型选择、MCP 服务器
- 典型场景：大型重构（Backend Agent + Frontend Agent + Test Agent 同时工作）

#### 核心能力三：工程化使用 - Skills / Hooks / MCP

- **Skills**：可复用的 "技能包"，包含 SKILL.md 描述文件和配套脚本模板
- **Hooks**：绑定到特定事件的确定性代码钩子，比如编辑后自动格式化、提交前自动 lint
- **MCP（Model Context Protocol）**：标准化的 AI 工具集成协议，可以连接 Google Drive、Jira、Slack、Figma 等外部数据源
- **Plugins**：把 Skills、Hooks、MCP Server 打包成可分发的插件单元，一键扩展能力

#### 关键数据

- 开发效率提升：**10 倍**
- Figma 到生产 UI：**1 小时**
- 单人完成完整模块交付：**1 个月**

### Feature 2: Claude Cowork - 桌面 AI 协作助手

- Claude Cowork 是 Anthropic 的桌面端 AI Agent，2026 年发布
- 可以把它理解为 **"办公场景的 Claude Code"**
- 它不是对话式 AI - 它是一个 **数字同事**，能独立执行多步骤任务
- 你描述一个目标，Claude 自动规划、分解任务、并行执行，把完成的文档/表格/报告直接保存到你的文件系统
- macOS 版 2026年1月16日上线，Windows 版 2月10日上线
- 需要 Claude Desktop + Pro/Max/Team/Enterprise 订阅

#### 核心能力

- **直接文件访问**：读写本地文件，不需要手动上传下载。直接分析桌面上的 Excel 并输出清洗后的文件
- **独立任务执行**：描述目标后独立完成。比如自动读取 10 份客户反馈文档，生成汇总 PPT
- **Sub-Agent 协调**：复杂任务自动分解并行执行 - 一个 Agent 分析数据，另一个生成图表
- **专业文档输出**：Excel 带 VLOOKUP/条件格式/多工作表、PPT 带正确排版、Word 带目录和页码

### Feature 3: 应用集成 - Claude in PPT, Excel & Documents

- Claude 可以深度集成到日常办公工具中，实现从数据分析到演示文稿的全流程自动化
- 通过 Claude Cowork 的桌面 Agent 能力和 MCP 协议，Claude 从聊天窗口走进了你熟悉的办公环境

#### 具体能力

- **PowerPoint 自动生成**：输入主题和大纲，自动生成完整 PPT - 布局、配色、图表占位符，支持企业模板和品牌色
- **Excel 深度分析**：理解数据结构、创建透视分析、复杂公式（VLOOKUP / INDEX-MATCH / 条件求和），输出带公式的文件

---

### Feature 4: Multi-Agent System - 多 Agent 协作

- 多个 AI Agent 协同工作，每个专注特定任务领域，协调完成复杂项目
- 这不是科幻 - **在 Claude Code 和 Claude Cowork 中已经是生产可用的功能**

#### 核心优势

- **并行处理**：多个 Agent 同时执行不同任务，大幅缩短完成时间（调研/财务分析/PPT 并行进行）
- **自动协调**：Team Lead Agent 负责任务分配、进度追踪、结果整合；Worker Agent 完成后自动汇报
- **复杂项目分解**：把大项目拆成独立子任务，用最合适的 Agent 执行，自动合并结果
- **专业分工**：每个 Agent 可以有不同的 Skills、工具权限、模型配置（Opus 做研究 / Sonnet 写文案 / Haiku 做校验）

---

### Feature 5: 工作流设计 - 将 AI 融入日常工作

- 最后一个高阶功能，也是最重要的思维方式转变
- **核心理念**：把重复性、规则化的任务交给 AI 自动化；人类专注需要判断力、创造力、关系建设的工作
- 好的 AI 工作流应该像水电一样 - **无处不在，但感知不到**

#### 落地方法

- **Prompt as Configuration**：把 Prompt 当作版本管理的配置文件，每个业务流程有标准化模板
- **Prompt 版本管理**：像代码一样记录变更、对比输出、回滚历史
- **人机协作流程**：明确 AI 独立完成部分、人工审核部分、人机协作部分（草稿 -> 审核 -> 修改 -> 终审）
- **持续改进闭环**：根据实际效果优化 Prompt、调整工作流节点、更新模型选择，形成反馈回路
